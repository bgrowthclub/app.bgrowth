# v0.1.0 — Platform Foundation

**Tagged:** `v0.1.0-platform-foundation` (commit `40376ec`, merged from `claude/engineering-foundation-p70xwe` into `main`)
**Status:** Released to `main` / production

## Overview

This is the first release of the BGrowth platform since `main` was frozen
at "Sprint 5 - Member Experience Foundation." It brings 40 commits of
accumulated work into `main` in a single, reviewed merge: the BGrowth
Workspace™ member shell, the Commerce and Identity architecture
foundations, a full homepage redesign, the purchase flow, the Member Area
/ Product Library, and BGrowth Studio's Product Engine together with its
live connection to the Runtime. This release went through a formal RC1
technical review (build, TypeScript, runtime errors, console errors,
broken links, routing, navigation, responsive behavior, accessibility,
dead code, unused imports, terminology, and performance) before merge —
see `docs/development/rc1-release-summary.md` and
`docs/development/rc1-review-checklist.md` for the full audit trail this
release was built from.

This app remains a **static, client-only MVP**: no backend, no database,
no real payment processing. Every "real" behavior described below (mock
authentication, mock purchases, mock product access) is genuinely working
client-side behavior, simulated with in-memory state and localStorage —
not a placeholder UI with no logic behind it, but also not backed by a
real server anywhere.

## Major Features

### BGrowth Workspace™ (Platform Shell)
The authenticated member shell members land in after login — sidebar,
top bar, global search, dashboard sections (Continue Your Workspace, My
Workspaces, Recently Added, Recommended, Quick Actions, Member Benefits).
Ecosystem products not built yet (Academy, Club, Marketplace, Find) are
visibly present but locked, with a "Coming Soon" modal instead of a dead
link.

### BGrowth Identity™
A real, working mock authentication system: Login, Register, Forgot
Password, Reset Password, Verify Email, and route protection
(`/platform/*` redirects a signed-out visitor to `/login`). No real
credential store or session token exists behind it — it's genuine
client-side behavior, not a real backend.

### BGrowth Commerce™
The provider-agnostic domain layer every future payment/membership/reward
integration will build on. Three services are real and working today:
`ProductService` (what the Runtime reads), `ProductAdminService` (what
Studio writes), and `AccessService` (who has access to what, independent
of how they got it). Everything else in Commerce (`PurchaseService`,
`MembershipService`, `PricingService`, `BenefitService`, `PartnerService`,
`RewardService`) is a real type-safe interface with mock data, not yet a
real implementation.

### Homepage, Checkout, and the Member Area
A full homepage narrative rebuild (goal-oriented Hero, Workspace™
ecosystem categories, a live "Start Here" browse row, Knowledge preview,
About). A working purchase-review flow (Checkout, Checkout Success) that
loads live product data rather than a static snapshot. A Member Area
("My Workspaces") showing exactly what the mock member owns, derived
through `AccessService` + `ProductService`, not a hardcoded list.

### BGrowth Studio — Product Engine
An internal Product Management tool at `/studio/products`: a 6-tab editor
(General, Content Source, Pricing, Images, Website, Publishing) covering
every commercial aspect of a product — pricing, images, SEO, categories,
visibility, related products — without ever editing the underlying
Workspace content itself. Includes a pluggable Content Source system (so
a future Planner, Calculator, or Academy course product type can plug in
without redesigning the tool), a consolidated media model
(`ProductAssets`), Draft/Published version history, and a Preview
capability that previews a draft product's content before it's published.

### The Runtime ↔ Product Engine connection
The centerpiece of this release: a **Published Product Repository** that
Studio's Publish action writes into, and that the entire public site reads
from exclusively. A product created and published in Studio appears on
the Homepage, the Workspace Catalog, its own dynamic Product Page, platform
Search, and Checkout **immediately** — no rebuild, no manual refresh, no
new React page or component required. This closes the loop the Product
Engine was originally built for.

## Architecture

- **Three top-level layouts**: `AppLayout` (public marketing),
  `PlatformLayout` (member Workspace), `StudioLayout` (internal Product
  Engine) — the third is a deliberate, documented exception to what was
  originally "exactly two layouts."
- **Provider-agnostic domain modules** (`modules/identity/`,
  `modules/commerce/`): a permanent interface layer separated from
  whatever concrete provider eventually implements it. Application code
  depends on the interface, never a provider SDK — no auth vendor or
  payment processor is wired in anywhere.
- **Read/write service separation**: `ProductService` (read-only, what
  the Runtime consumes) and `ProductAdminService` (write, what Studio
  uses) are two distinct implementations sharing one underlying mock data
  store.
- **Published Product Repository as the publish boundary**: Studio's
  draft working store and the Published Product Repository are separate
  in-memory stores. Publishing is the only action that copies a product
  from one to the other — further edits after publishing don't go live
  until Publish is clicked again.
- **Governing documents**: `CLAUDE.md`, `ARCHITECTURE.md`, `VISION.md`,
  and `PRODUCT_CATALOG.md` are now the permanent reference set for how
  this codebase is built and why, including formal rules like the
  Component Evolution Rule and "never delete a component without
  explicit approval."
- **Feature-branch workflow adopted going forward** — see
  `docs/development/branching-strategy.md`. This release is also the
  point where the long-running `claude/engineering-foundation-p70xwe`
  branch is retired in favor of one branch per feature.

## Breaking Changes

Relative to what `main` had before this release:

- **`/industries` route removed** (and `IndustriesPage.tsx`,
  `data/industries.ts` deleted) — replaced by `/workspaces` and
  `/preview/:category`. Any external link to the old route now resolves
  to the new 404 page (see Known Limitations) instead of a working page.
- **Several homepage sections removed outright**, not kept alongside
  their replacements: `BecomeMember`, `FeaturedSystems`, `HowItWorks`,
  `Industries`, `Testimonials`.
- **`Navbar`, `Footer`, `Hero`, `SEO` rewritten** — structural and visual
  changes to every page rendered inside `AppLayout`.
- **The app is now wrapped in `MockIdentityProvider`** (`main.tsx`), and
  **`/platform/*` is now gated** behind a mock session via
  `ProtectedRoute` — a signed-out visitor is redirected to `/login`,
  which did not happen before this release.
- **`data/systems.ts` gained new required accessors**
  (`getAllSystems`, `getRecentlyAddedSystems`, `getRecommendedSystems`,
  `searchSystems`, `getOwnedSystems`) — code reading `SYSTEMS` directly
  needs to move to an accessor.

## Known Limitations

- **No real backend of any kind.** No database, no real authentication
  provider, no real payment processor. Every "real" behavior described
  above is genuine client-side logic, not connected to a server.
- **No GitHub, Stripe, or BGrowth Studio (external repo) integration
  yet.** The Published Product Repository is in-memory only; it resets on
  every full page reload. A future `RemoteProductRepository` backed by a
  real Remote Product Source (GitHub, S3, an API) is designed for but not
  built.
- **BGrowth Studio's Product Engine has no auth gate.** `/studio/*` is
  reachable by anyone who knows the URL — there is no real staff/admin
  authentication yet. This is a known, intentional gap, not an oversight.
- **Most of Commerce is interface-only.** `PurchaseService`,
  `MembershipService`, `PricingService`, `BenefitService`,
  `PartnerService`, and `RewardService` are typed and mocked but have no
  real implementation — nothing in the app currently depends on them
  doing real work.
- **Only one Content Source provider exists** (GrowthSystem/Workspace).
  Planner, Calculator, Academy Course, and Template Pack are reserved
  types in the Product Engine's Content Source picker but show as
  "coming soon" — no provider has been built for them yet.
- **Five files are intentionally unused, not deleted**, per this repo's
  no-silent-deletion policy: `components/systems/OwnedSystemCard.tsx`,
  `studio/components/tabs/WorkspaceTab.tsx`,
  `studio/lib/workspaceImport.ts`,
  `mock/mockProducts.ts`'s `createLocalProductRepository`, and the dormant
  `src/content/` scaffold. All are safe to remove pending explicit
  approval.
- **No test suite and no linter are configured.** Every verification in
  this release was manual (`npm run build`, `tsc --noEmit`, and hands-on
  browser verification) — there is no automated regression coverage.
- **Only one mock member exists**, and Studio's Product Engine has no
  concept of multiple staff users — this remains a single-user simulation
  throughout.
- **A catch-all 404 page was added in this release** (`NotFoundPage`,
  registered as `path="*"`) specifically because none existed before —
  every previously-unmatched URL rendered a blank page.

## Future Roadmap

Not committed to specific sprints — for planning discussion once the next
Sprint is scoped:

- **Real provider implementations** for BGrowth Identity™ (a real auth
  vendor) and BGrowth Commerce™ (a real payment processor, e.g. Stripe) —
  the provider-agnostic interfaces this release built are designed
  specifically so this is a swap-in, not a rewrite.
- **A real Remote Product Source** (GitHub, S3, or an API) backing the
  Published Product Repository, so publishing survives a page reload and
  can eventually be triggered from a separate BGrowth Studio repository
  rather than this app's internal `/studio/*`.
- **Additional Content Source providers** (Planner, Calculator, Academy
  Course, Template Pack) so Studio's Product Engine can wrap more than
  just Workspace content.
- **Additional Growth Categories** beyond Business & Entrepreneurship
  (Careers, Languages, Personal Finance, Productivity, Education, Health
  & Wellness, Family & Lifestyle) — architecturally a data change, not a
  new Runtime or page pipeline, per `ARCHITECTURE.md`.
- **The remaining ecosystem products** (Academy, Club, Marketplace, Find,
  AI, Partners, Rewards, Benefits) — currently locked placeholders in the
  Workspace sidebar.
- **A real auth gate for BGrowth Studio**, once real staff/admin
  authentication exists.
- **Removal of the five intentionally-unused files** listed under Known
  Limitations, once explicitly approved.
